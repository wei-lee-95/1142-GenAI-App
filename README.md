# 長照 2.0 機構補助諮詢助手

這個專案使用 RAG（Retrieval-Augmented Generation）技術，將長照 2.0 相關補助辦法 PDF 建立成可檢索的知識庫，並透過 Gemini 回答長照機構補助申請相關問題。系統會先從官方文件中找出最相關的條文片段，再把片段提供給 Gemini 生成答案，以降低模型憑空回答的機率。

## 專案動機

台灣長照機構數量多，且政府補助項目、資格條件、申請期限與金額規定分散在不同條文中。對機構服務人員來說，逐條查詢文件會花費不少時間。

本專案希望提供一個補助諮詢助手，協助使用者更快查詢：

- 補助項目與金額
- 申請資格與限制
- 申請期限
- 特定地區或身分條件下的補助規則
- 條文出處與依據

## 資料來源

- 官方資料集：[衛生福利部長照補助相關文件](https://www.mohw.gov.tw/dl-92881-b226ce9a-7a75-4a5b-b810-714f2ff37c0a.html)

## 使用技術

- Python
- Google Gemini API
- LangChain
- FAISS
- PyPDF2
- RAG

主要流程：

1. 讀取長照補助 PDF。
2. 使用 `RecursiveCharacterTextSplitter` 將文件切成小段落。
3. 使用 `gemini-embedding-001` 將文字轉成 embedding。
4. 使用 FAISS 建立向量資料庫。
5. 使用者提問時，先做 similarity search 找出相關條文。
6. 將檢索到的內容作為 context，交給 `gemini-2.5-flash` 產生回答。

## 評估問題

專案中使用以下五個問題比較一般 Gemini 回答與 RAG 系統回答：

1. 「營養餐飲服務」每餐補助多少錢？一個人一天最多幾餐？
2. 我想申請「身心障礙者日間照顧機構費用」，最晚什麼時候要申請？
3. 我是中低收入戶，失智等級 CDR 2 分，住進團體家屋一個月補助多少？
4. 如果我在「桃園市復興區」當居服員，照顧 3 個以上的個案，每月有多少獎勵津貼？
5. 我家長輩失智併有 BPSD 症狀，在申請補助時有什麼要注意的？

實驗觀察：

- 一般 Gemini 有時會提供較模糊的範圍或額外資訊。
- 本專案的 RAG 回答會根據 PDF 內容回答。
- 回答可附上較明確的文件依據，降低幻覺風險。

## 結論

本專案成功建立一個基於 RAG 的長照機構補助諮詢助手。透過官方文件檢索與 Gemini 生成回答，系統能更精準地回應補助相關問題，協助機構服務人員減少查詢條文的時間。
